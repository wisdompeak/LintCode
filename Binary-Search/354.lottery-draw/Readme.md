### 354. lottery draw

#### Description
```
小明最近在研究一款游戏。在游戏中，有一些东西可以通过刷刷刷获得，有一些东西只能通过抽奖获得。
抽奖存在一个这样的机制：单次抽奖，抽到好东西的概率是a%。
当连续b-1次未抽到好东西时，从第b抽开始，每一次抽到好东西的概率会比前一次的概率提升c%，直到100%。

游戏开发商使用这样的方法降低抽到物品需要的期望抽奖次数，并用期望抽奖次数的倒数p%冒充单抽抽到好东西的概率a%。
小明知道了(b,c,p)的值，希望计算出a。你能帮助他计算出结果吗？

返回一个浮点数，表示a，系统将保留小数点后2位。

在第b-1轮时抽到好东西的概率为a%。在第b轮时抽到好东西的概率为a%+c%。

数据保证b,c,p都是整数，1≤c,p≤100,2≤b≤1000。保证数据合法。
```

#### 分析：
```
本题的本质就是求a，使得抽中奖品的次数的期望是 e0 = 1/p%.

可以有直观的感觉: a越大，越容易抽中，那么抽中奖品的次数的期望越小。反之，a越小，越不容易抽中，那么抽中奖品的次数的期望越大。
所以这一个目标明确的二分搜索：猜一个a的值，计算期望e，查看与e0的大小关系，然后调整a的搜索范围。

什么是“抽中奖品的次数的期望”？
e = sum {在第i次抽中奖品的概率 * i }
根据题目，抽中奖品的概率迟早会上升到100%，因此e的计算是有限项的相加，可以暴力计算。
（PS，只要级数收敛，期望的计算也可以是无限项的相加）

如何高效计算“在第i次抽中奖品的概率”？
在第i次抽中奖品的概率 = p_unluck[i-1] * p_luck[i]
前者是前i-1轮，轮轮不中的概率；后者是第i轮抽中的概率 p_luck[i] = a, a, a, ..., a, a+c, a+2c, a+3c, ....

记得更新 p_unluck[i] = p_unluck[i-1] * (1 - p_luck[i])
```
